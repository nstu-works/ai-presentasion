**Статус:** На ревью

---

## 1. Контекст и цель

Живой доклад (45–60 минут) для аудитории разработчиков. Цель — поделиться личным опытом вайбкодинга: какие AI-инструменты используются, как выстроен рабочий флоу, какие скиллы упрощают жизнь. Презентация не является инструкцией — это личный опыт докладчика.

**Технология:** Reveal.js (Подход B — разделённые файлы, локальный сервер через npm).  
**Язык слайдов:** смешанный (заголовки и основной текст на русском, технические термины — на английском).  
**Аудитория:** разработчики (базу объяснять не нужно).

- Акцент: промт для security стареет быстрее, чем его успеваешь поддерживать

### Раздел 9 — Фишки (7 слайдов)

**9.0 — Память проекта**
- Файл на диске: что сохраняется, где лежит
- Персистентность между сессиями — нейронка помнит контекст проекта

**9.1 — Управление моделями**
- Haiku → рутинные задачи (дёшево, быстро)
- Opus → сложные задачи (мощно, дороже)
- Автоматическое переключение через настройки

**9.2 — SDD + TDD**
- 1 задача = 1 агент
- Задача закрывается когда тесты зелёные
- Авто-ревью после завершения
- Результат: предсказуемое качество без ручного контроля каждого шага

**9.3 — Параллельные агенты**
- `/dispatching-parallel-agents`
- Схема: главный агент → N параллельных агентов → сборка результатов

**9.4 — Пример: шпаргалка по 12 лекциям**
- Задача: выписать ключевые знания из 12 лекций
- Флоу:
  - 12 агентов запущены параллельно
  - Каждый агент обрабатывает одну лекцию → записывает в файл
  - Когда все готовы — главный агент собирает все файлы в один
- Визуал: диаграмма fan-out / fan-in
- Заметки докладчика: привести реальное время выполнения vs последовательно

**9.5 — MCP серверы: что это**
- MCP (Model Context Protocol) — стандарт для подключения внешних инструментов к AI
- Схема: Claude → MCP-сервер → внешний источник (база данных, API, документация, файловая система)
- Как подключить: одна строчка в настройках Claude Code → Claude видит новый инструмент
- Ключевая идея: «Ты расширяешь мозг нейронки подключаемыми источниками данных»
- Заметки докладчика: объяснить разницу — без MCP нейронка знает только то, что в тренировочных данных; с MCP — она может запросить актуальные данные в реальном времени

**9.6 — Context7: MCP для документации**
- Что делает: когда Claude пишет код, Context7 автоматически подтягивает актуальную документацию нужной библиотеки
- Проблема без него: нейронка знает устаревшие API из обучающей выборки → генерирует deprecated-код
- С Context7: «Напиши мне запрос на Prisma» → Claude спрашивает Context7 → получает доку текущей версии → пишет правильно
- Крупно: «Никаких галлюцинаций из-за устаревших знаний»
- Заметки докладчика: Context7 бесплатный, ставится одной командой, поддерживает тысячи популярных библиотек
